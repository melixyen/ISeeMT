# 快速開始指南

## 方式一：使用獨立 HTML 版本（最簡單）

### 步驟
1. 開啟 `standalone/index.html` 檔案（用瀏覽器直接開啟）
2. 按 F11 進入全螢幕模式
3. 使用右側控制面板選擇測試圖案
4. 開始測試！

### 優點
- 不需要安裝任何東西
- 跨平台（任何有瀏覽器的系統）
- 立即可用

### 缺點
- 無法自動偵測多螢幕
- 需要手動將瀏覽器視窗拖到要測試的螢幕

## 方式二：建置 Tauri 應用程式（完整功能）

### 前置需求
1. **安裝 Node.js** (18 或更新版本)
   - 下載：https://nodejs.org/
   - 驗證：`node --version`

2. **安裝 Rust** (1.70 或更新版本)
   - 下載：https://rustup.rs/
   - 驗證：`rustc --version`

3. **Windows 使用者需要安裝 Visual Studio Build Tools**
   - 下載：https://visualstudio.microsoft.com/downloads/
   - 選擇「Desktop development with C++」工作負載

### 安裝步驟

#### 1. 安裝依賴
```bash
cd ISeeMT
npm install
```

#### 2. 開發模式（推薦用於測試）
```bash
npm run tauri:dev
```

這會啟動開發版本，可以即時看到程式碼變更。

#### 3. 建置正式版本
```bash
npm run tauri:build
```

建置完成後，執行檔位於：
- `src-tauri/target/release/iseemt.exe` (Windows)
- 安裝包位於 `src-tauri/target/release/bundle/`

### 使用方式

1. **啟動程式**
   - 執行 `iseemt.exe` 或使用 `npm run tauri:dev`

2. **選擇螢幕**
   - 程式會自動偵測所有連接的螢幕
   - 點擊要測試的螢幕按鈕

3. **選擇測試圖案**
   - 使用右側控制面板選擇測試圖案
   - 或按空白鍵隱藏控制面板

4. **快捷鍵**
   - `ESC`: 返回螢幕選擇器
   - `空白鍵`: 顯示/隱藏控制面板

## 測試圖案說明

### 靜態測試

#### 純色測試
- **用途**: 檢查螢幕是否有亮點、暗點、色偏
- **測試項目**: 白色、黑色、紅色、綠色、藍色
- **如何測試**: 
  1. 選擇純色圖案
  2. 仔細觀察螢幕是否有異常點
  3. 黑色背景最容易看出亮點
  4. 白色背景最容易看出暗點

#### 灰階測試
- **用途**: 檢查螢幕灰階顯示能力
- **測試項目**: Gray (128)、Gray (204)
- **如何測試**:
  1. 觀察灰階是否均勻
  2. 檢查是否有色偏（應該是純灰色）

#### 色階表
- **用途**: 檢查螢幕色彩漸層顯示能力
- **如何測試**:
  1. 觀察 RGB 三色漸層是否平滑
  2. 檢查是否有斷層或跳色
  3. 底部 16 級色階應該清晰可辨

#### 線條測試
- **用途**: 檢查螢幕像素對齊和清晰度
- **測試項目**: 水平線、垂直線、十字線
- **如何測試**:
  1. 觀察線條是否清晰
  2. 檢查是否有模糊或重影
  3. 十字線可檢查九宮格對齊

#### 均勻性測試
- **用途**: 檢查螢幕亮度均勻性
- **測試項目**: 白色、黑色、紅色、綠色、藍色
- **如何測試**:
  1. 觀察五個方塊（中央+四角）亮度是否一致
  2. 檢查背景是否有漏光
  3. 特別注意四個角落

### 動態測試

#### 灰階波動
- **用途**: 檢查螢幕灰階過渡能力
- **如何測試**:
  1. 觀察灰階變化是否平滑
  2. 檢查是否有跳階或閃爍

#### 方塊動畫
- **用途**: 檢查螢幕動態響應
- **如何測試**:
  1. 觀察方塊移動是否流暢
  2. 檢查是否有殘影或拖影

#### 圓形動畫
- **用途**: 檢查螢幕圓形顯示能力
- **如何測試**:
  1. 觀察圓形是否完美
  2. 檢查縮放是否平滑

#### 閃爍測試
- **用途**: 檢查螢幕閃爍問題
- **如何測試**:
  1. 觀察黑白切換是否乾淨
  2. 檢查是否有殘影

### 進階測試

#### 亮度堆疊
- **Light Stack**: 高亮度階層 (207-255)
- **Med Stack**: 中等亮度階層 (128-192)
- **Dark Stack**: 低亮度階層 (0-48)

**如何測試**:
1. 觀察能分辨出幾個階層
2. 數字越多表示螢幕階調越好
3. 如果無法分辨所有階層，可能需要調整螢幕設定

## 常見問題

### Q: 獨立版本可以測試多螢幕嗎？
A: 可以，但需要手動將瀏覽器視窗拖到要測試的螢幕，然後按 F11 全螢幕。

### Q: 為什麼建置失敗？
A: 請確認：
1. Node.js 和 Rust 已正確安裝
2. Windows 使用者已安裝 Visual Studio Build Tools
3. 執行 `npm install` 安裝所有依賴

### Q: 如何測試特定顏色？
A: 使用「Custom Color」功能，輸入 RGB 值（0-255），然後點擊 Apply。

### Q: 動畫太快或太慢？
A: 在控制面板的「Dynamic Tests」區域調整 Speed 滑桿。

### Q: 如何退出全螢幕？
A: 
- Tauri 版本：按 ESC
- 獨立版本：按 ESC 或 F11

## 推薦測試流程

### 新螢幕檢測流程
1. **亮點/暗點檢測**
   - 黑色背景（檢查亮點）
   - 白色背景（檢查暗點）

2. **色彩檢測**
   - 紅色、綠色、藍色（檢查色偏）
   - 色階表（檢查色彩過渡）

3. **均勻性檢測**
   - 白色均勻性（檢查漏光）
   - 黑色均勻性（檢查背光均勻度）

4. **動態響應**
   - 方塊動畫（檢查殘影）
   - 灰階波動（檢查過渡）

5. **階調能力**
   - Light Stack（高亮階調）
   - Dark Stack（暗部階調）

### 日常維護檢測
1. 白色背景（快速檢查暗點）
2. 黑色背景（快速檢查亮點）
3. 灰階測試（檢查色偏）
